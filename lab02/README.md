### Метод конечных разностей для уравнения теплопроводности

**Задание:**  
Реализовать моделирование изменения температуры в пластине на основе одномерного уравнения теплопроводности с использованием метода конечных разностей.

Выполнить моделирование с различными шагами по времени и по пространству.  
Заполнить таблицу значений температуры в центральной точке пластины после 600 секунд модельного времени.

#### Таблица результатов моделирования

| Шаг по времени, с \ Шаг по пространству, м | 0.1 | 0.01 | 0.001 | 0.0001 |
|-------------------------------------------|-----|------|-------|--------|
| **0.1** | 24.58 | 21.40 | 21.37 | 21.37 |
| **0.01** | 24.58 | 21.40 | 21.37 | 21.37 |
| **0.001** | 24.58 | 21.40 | 21.37 | 21.37 |
| **0.0001** | 24.58 | 21.40 | 21.37 | 21.37 |

#### Таблица времен моделирования

| Шаг по времени ($dt$), с \ Шаг по пространству ($h$), м | 0.1 | 0.01 | 0.001 | 0.0001 |
|:-------------------------------------------|:-----:|:------:|:-------:|:--------:|
| **0.1** | 0.3923* | 0.0025 | 0.0166 | 0.1499 |
| **0.01** | 0.0016 | 0.0156 | 0.1481 | 1.5023 |
| **0.001** | 0.0159 | 0.1523 | 1.4656 | 14.7583 |
| **0.0001** | 0.1569 | 1.5562 | 14.7704 | 150.6628 |


**Вывод:**
* Есть явная зависимость точности от шага сетки. При грубом шаге $h=0.1$ значение температуры 24.58 заметно завышено. При уменьшении шага до $0.001$ решение стабилизируется на значении 21.37. Дальнейшее измельчение до $0.0001$ не изменяет результат, что свидетельствует о достижении сеточной сходимости
* Использование метода прогонки обеспечило абсолютную устойчивость решения. Значения температуры идентичны при разных $dt$ для одного и того же $h$. Это показывает, что для данной задачи шаг $dt=0.1$ является достаточным

Также можно заметить, что в таблице времен аномально высокое значение в самой первой ячейке, это связано с тем, что я использовал JIT библиотеку, которая (как и другие JIT библиотеки по типу jax, cpython, numba и др.) при первом запуске программы прогоняет трассировочные данные, чтобы записать, например, размерности и скомпилировать программу, в остальном время полностью соответствует трудоемкости O($N_x$ * $N_t$)
